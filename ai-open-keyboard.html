Privacy-First AI Keyboard

Open Keyboard

An iOS keyboard prototype for AI-assisted writing, with grammar correction and broader writing actions available directly inside the custom keyboard extension.

Open Keyboard is the client experience: a custom iOS keyboard where you can type normally and ask for AI help when you need grammar fixes, improvements, rephrasing, or summaries.

The privacy direction is that normal typing stays on the device. Text is sent out only when the user chooses an AI action, and it goes to the gateway they configure.

Current status: active prototype. Recent work focused on the keyboard extension itself: separating grammar review from AI writing tools, making Improve perform a real request, and tightening reliability around loading and replacement behavior.

Latest Product Progress

Writing

AI Writing Tools In The Keyboard

The top-right AI action button now opens a writing workflow instead of a static mock, with source text, generated suggestions, selected action state, and explicit apply behavior.

  • Improve runs through the configured AI path.
  • Users can retry a request, show or hide action options, copy a suggestion, or accept it into the current text.
  • Rephrase and Summarize remain part of the same writing-action surface.
Grammar

Separate Grammar Correction

The left grammar/status control is now treated as its own workflow for typo and grammar review, rather than being mixed with the broader writing tools.

  • Tapping it starts or shows grammar correction for the current text.
  • Results can show corrections, a no-issues state, or a visible error state.
  • The control stays available unless there is a hard blocker such as missing Full Access, missing gateway setup, a gateway connection error, or an unrecovered AI/parsing error.
Quality

More Reliable Correction Flow

Manual grammar correction is tracked separately from automatic analysis, which prevents a cancelled background analysis from leaving the manual correction flow stuck in loading.

  • Manual grammar requests have their own request state.
  • Correction replacement respects word boundaries for single-character fixes.
  • Article corrections such as changing "a" to "an" avoid stale replacements that could duplicate text.
Coverage

Deterministic Preview And Tests

The preview lab and regression tests were updated for the new Improve UI and the corrected grammar workflow behavior.

  • Preview support covers the new AI writing panel states.
  • Regression tests cover grammar loading and correction replacement behavior.
  • Focused real-extension smoke coverage verifies the configured keyboard AI controls.
