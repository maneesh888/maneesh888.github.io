Self-Hosted AI Backend

Open Keyboard LLM Gateway

A small gateway that connects Open Keyboard-style clients to local or OpenAI-compatible models behind one controlled API.

This project supports Open Keyboard, where the keyboard handles the writing experience and the gateway controls where AI requests go, which models are available, and who is allowed to use them.

The gateway checks the app key, shows available AI models, applies usage limits, and sends the request to the AI model the user chooses.

What this demonstrates: practical backend architecture for self-hosted AI tools, including auth, model access control, admin operations, OpenAI-compatible endpoints, safer setup, and tests around critical API behavior.

Technical Highlights

Auth

API Key Authentication

Validates app keys before model requests reach the proxy layer, with admin flows for creating and managing access.

Limits

Rate Limiting

Applies usage limits per app key so keyboard clients and experiments can share the backend without uncontrolled traffic.

Models

Model Routing

Routes OpenAI-compatible chat requests to local or compatible model backends without changing the keyboard integration pattern.

Admin

Admin Operations

Includes an admin path for key management, model configuration, and operational control of client access.

Setup Flow

01

Start The Gateway

Run the server that connects the keyboard to the chosen AI model.

02

Protect The Admin Page

Set an admin password and secret before opening the gateway to anyone else.

03

Create An App Key

Create an API key for the keyboard, then choose which models it can use and how many requests it can send.

04

Connect The Keyboard

Enter the gateway URL and API key in Open Keyboard, then test the connection.

05

Enable The iOS Keyboard

Turn on Open Keyboard in iOS Settings. Full Access is needed only for AI actions that call the gateway.

06

Use The Writing Tools

Choose Fix Grammar, Rewrite, or Summarize from the keyboard. Normal typing stays local.
